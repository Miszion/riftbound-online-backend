# Riftbound Replay Viewer

Standalone Next.js 15 app that renders a Riftbound self-play match from a JSONL
event log, frame by frame, at 1920x1080 with large commentary overlay so the
output can be captured as video.

This app is independent of the backend - it reads JSONL files served as static
assets from `public/matches/`.

## Getting started

```bash
# From repo root:
cd apps/game-viewer

# The shared shell has NODE_ENV=production which prunes devDeps, so we have to
# override it to install Tailwind, TypeScript, etc.
NODE_ENV=development npm install --include=dev

# Populate public/matches/ with .jsonl files (see below), then:
NODE_ENV=development npm run dev
# -> http://localhost:4200/
```

The `dev` and `build` scripts automatically regenerate `public/manifest.json`
from `public/matches/`.

## Loading matches

Drop the bot self-play `.jsonl` files into `public/matches/`. For example:

```bash
cp /Users/miszion/workplace/nexus-data/riftbound-games/*.jsonl \
   apps/game-viewer/public/matches/
npm run build-manifest
```

The homepage lists every match in the manifest. Clicking one opens
`/replay?match=<filename>`.

## URL parameters

| Param       | Default | Description                                                 |
| ----------- | ------- | ----------------------------------------------------------- |
| `match`     | -       | Filename inside `public/matches/` (required).               |
| `autoplay`  | `1`     | Auto-play on load. Set to `0` to start paused.              |
| `chain`     | -       | If `next`, auto-advances to the next match on MATCH OVER.   |

## Keyboard shortcuts

- `Space` - play/pause
- `Arrow Right` - step forward
- `Arrow Left` - step back

## Layout (1920x1080)

- Top: phase banner + VP/HP/mana scoreboard
- Middle left: battlefield grid (color-coded by controller)
- Middle right: P1 and P2 boards with hand/deck/board/graveyard counts
- Bottom (48pt font): commentary for the current action
- Bottom bar: play/pause, step, speed (1x/2x/4x), scrubber
